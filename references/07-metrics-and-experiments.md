# 指标、埋点与实验

## 指标设计原则

指标要服务于决策，而不是为了看起来完整。每个指标都要说明用途：验证假设、监控风险、发现问题或指导迭代。

## 指标类型

| 类型 | 作用 |
|---|---|
| 北极星指标 | 衡量长期核心价值 |
| 业务结果指标 | 收入、转化、留存、成本、效率 |
| 用户行为指标 | 曝光、点击、使用、完成、复访 |
| 过程指标 | 漏斗每一步转化 |
| 体验指标 | 时长、错误率、满意度、投诉率 |
| 护栏指标 | 防止局部优化伤害整体体验 |
| 反指标 | 发现滥用、打扰、误触、负反馈 |

## 指标树

输出指标时按层级：
1. 核心目标
2. 一级指标
3. 二级拆解
4. 可观测事件
5. 数据来源
6. 判断阈值

## 埋点设计

埋点表包含：
- 事件名
- 触发时机
- 事件属性
- 用户属性
- 页面/入口
- 用途
- 负责人/数据来源（如已知）

## A/B 实验设计

实验方案需要包含：
- 核心假设
- 实验对象
- 对照组
- 实验组
- 分流方式
- 实验周期
- 主指标
- 护栏指标
- 样本量或样本范围
- 成功标准
- 失败后的决策规则

## 非实验验证

如果无法 A/B，可使用：
- 灰度发布
- 小范围试点
- 人工服务验证
- 用户访谈
- 可用性测试
- 数据前后对比
- 运营活动验证
