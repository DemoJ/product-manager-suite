# HTML 原型设计

## 定位与原则

HTML 原型是产品经理在方案设计阶段的**可交互可视化输出**。它不是设计稿，而是用 HTML/CSS/JS 直接构建的可点击、可操作的产品 demo。

### 核心原则

1. **黑白灰优先**：只用灰度色阶，不引入彩色。目的是让评审者聚焦信息架构、交互流程和功能逻辑，而不是视觉风格。唯一例外：用红色系标注错误/危险状态，用绿色系标注成功状态——这两者属于语义色，不是装饰色。
2. **可交互 demo**：所有原型都是可直接在浏览器打开使用的 demo，不是静态截图。按钮可点击、Tab 可切换、弹窗可展开、表单可填写、页面可跳转。
3. **平台匹配需求**：需求说移动端就画移动端，需求说桌面端就画桌面端，需求说双端就分别画。不默认、不猜测。
4. **组件不受限**：HTML 本身就是组件库。常见组件用成熟模式实现，不常见组件当场用 HTML/CSS 构建。不需要维护固定组件清单。
5. **原型不带标注**：交互说明、业务规则、边界情况写在配套的 Markdown 文档中，不叠加在原型页面上。原型保持干净。
6. **单文件交付**：每个原型是一个独立 HTML 文件，CSS 和 JS 全部内联，无外部依赖，双击即可在浏览器打开。

## 设计 Token 系统

所有原型共享统一的 Token 系统，确保风格一致。

### 灰度色阶

```
文字层级：
  #1a1a1a  页面主标题、核心数据
  #333333  正文、按钮文字
  #666666  次要说明、辅助信息
  #999999  占位符、禁用文字、时间戳

线条与边框：
  #d9d9d9  输入框边框、分割线
  #e8e8e8  卡片边框、弱分割线
  #f0f0f0  表格线、极弱分隔

背景层级：
  #f5f5f5  页面底色（区分白色容器）
  #fafafa  卡片/容器背景
  #ffffff  输入框、弹出层背景

交互灰度：
  按钮默认 #333 → hover #555 → active #1a1a1a
  次级按钮 边框 #d9d9d9 → hover 背景 #f5f5f5
  选中态：边框 #333 + 背景 #f0f0f0
  禁用态：文字 #c0c0c0 + 背景 #f5f5f5 + cursor: not-allowed
```

### 语义色（仅用于状态反馈）

```
  成功/正向  #52c41a  成功提示、通过状态
  错误/危险  #ff4d4f  表单校验错误、删除确认
  警告       #faad14  注意提示、待处理状态
```

### 字体

```
字体族：system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", sans-serif
字号层级：
  24px / 600  页面大标题
  20px / 600  区块标题
  16px / 600  卡片标题、列表标题
  14px / 400  正文、按钮文字
  12px / 400  辅助说明、标签、时间戳
行高：1.5（正文）、1.3（标题）
字重：400（常规）、600（标题/强调）
```

### 间距（4px 基准）

```
  4px   紧凑元素内间距（标签内padding）
  8px   列表项间距、小组件间距
  12px  卡片内 padding、表单项间距
  16px  区块间距、按钮组间距
  24px  模块间距、页面边距（移动端）
  32px  大区块分隔
  48px  页面顶部/底部留白
```

### 圆角

```
  4px   按钮、输入框、标签
  8px   卡片、弹窗、下拉面板
  12px  底部弹出层顶部
  50%   头像
```

### 阴影

```
  轻阴影：0 1px 4px rgba(0,0,0,0.08)  —— 卡片、浮动按钮
  中阴影：0 4px 12px rgba(0,0,0,0.12) —— 弹窗、下拉面板
  重阴影：0 8px 24px rgba(0,0,0,0.16) —— 抽屉、全屏弹出层
```

## 平台适配

### 移动端原型

```
画布宽度：375px（居中显示，两侧留灰底）
导航模式：顶部导航栏（44px）+ 底部 Tab 栏（50px）
页面边距：左右 16px
安全区域：底部预留 34px（iPhone Home Indicator）
手势提示：用文字标注代替手势图标（如 "[左滑删除]"）
字体：不缩放，保持 14px 正文基准
```

### 桌面端原型

```
画布宽度：1440px（或 1280px，根据需求）
导航模式：左侧边栏（200-240px）或顶部导航（56px）
内容区域：最大宽度 1200px 居中，或自适应
页面边距：左右 24-32px
最小高度：内容区不低于 900px
```

### 平板端原型

```
画布宽度：768px（竖屏）或 1024px（横屏）
导航模式：可折叠侧边栏或顶部导航
页面边距：左右 24px
```

## 交互模式

原型中的所有交互都用原生 HTML/CSS/JS 实现，不依赖任何框架。

### CSS 纯交互（优先使用）

```
hover 效果：按钮变色、卡片浮起、链接下划线
active 效果：按钮按下态
focus 效果：输入框高亮边框
transition：所有状态切换加 0.2s ease 过渡
checkbox/radio：用 CSS 自定义样式实现选中态
details/summary：原生折叠面板
:target 伪类：用锚点实现简单的弹窗/Tab 切换
```

### JS 交互（CSS 无法实现时使用）

```
页面跳转：单文件内用 section + scrollIntoView 模拟
          多文件用 location.href 跳转
弹窗/Modal：点击按钮显示/隐藏 overlay
抽屉/Drawer：侧边滑出面板
Tab 切换：点击切换内容区
折叠/展开：手风琴式内容折叠
表单联动：选择项变化时更新关联字段
列表操作：新增/删除/排序列表项
搜索筛选：实时过滤列表内容
Toast 提示：临时消息提示后自动消失
确认对话框：删除等危险操作的二次确认
拖拽排序：drag and drop API
```

### 多页面原型

当原型包含多个页面时，有两种方式：

**方式一：单文件多 section（推荐，≤10 个页面）**
```html
<!-- 每个页面是一个 section -->
<section id="page-list" class="page active">...</section>
<section id="page-detail" class="page">...</section>
<section id="page-form" class="page">...</section>

<!-- JS 控制页面切换 -->
<script>
function showPage(id) {
  document.querySelectorAll('.page').forEach(p => p.classList.remove('active'));
  document.getElementById(id).classList.add('active');
}
</script>
```

**方式二：多文件跳转（>10 个页面或需要独立分享）**
```
每个页面一个 HTML 文件，用 <a href="detail.html"> 跳转。
所有文件共享同一套 CSS 变量（内联在各自文件中）。
```

## 代码规范

### 文件结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>原型名称 - 页面名</title>
  <style>
    /* 1. Token 变量 */
    :root { ... }
    /* 2. 基础重置 */
    * { margin: 0; padding: 0; box-sizing: border-box; }
    body { ... }
    /* 3. 布局组件 */
    /* 4. 通用组件 */
    /* 5. 页面特定样式 */
  </style>
</head>
<body>
  <!-- 页面内容 -->
  <script>
    // 交互逻辑
  </script>
</body>
</html>
```

### 编码要求

```
1. 语义化 HTML：用 header/nav/main/section/footer 而不是全 div
2. 中文内容：所有文案用中文，贴近真实产品
3. 真实数据感：列表至少 3-5 条数据，不要只有 1 条
4. 空状态：列表为空时展示空状态设计
5. 边界情况：长文本截断、数字过大、极端值
6. 可访问性：按钮用 <button>，链接用 <a>，表单用 <label>
7. 代码注释：关键区块加注释说明用途
```

## 配套文档

每个原型配一份 Markdown 说明文档，结构如下：

```markdown
# [功能名称] 原型说明

## 原型文件
- 文件名：xxx.html
- 平台：移动端 / 桌面端 / 双端
- 页面数量：N 个

## 页面清单
| 页面 | 文件/锚点 | 说明 |
|---|---|---|
| 列表页 | #page-list | 展示 xx 数据列表 |
| 详情页 | #page-detail | 展示 xx 详细信息 |

## 交互说明
### 页面 A → 页面 B
- 触发方式：点击列表项
- 跳转逻辑：携带列表项 ID
- 返回方式：左上角返回按钮

### 弹窗：确认删除
- 触发方式：点击删除按钮
- 内容：确认文案 + 取消/确认按钮
- 确认后：列表刷新，Toast 提示"已删除"

## 业务规则
- 权限：仅管理员可见删除按钮
- 排序：默认按创建时间倒序
- 分页：每页 20 条，支持加载更多

## 边界情况
- 列表为空：展示空状态 + 引导创建
- 名称超长：最多显示 20 字，超出省略号
- 网络异常：展示重试按钮

## 待确认问题
- [ ] 是否需要批量操作？
- [ ] 搜索是否支持模糊匹配？
```

## 工作流程

1. **确认平台**：从需求中识别目标平台（移动端/桌面端/平板/多端）。
2. **梳理页面**：列出所有需要的页面和它们之间的跳转关系。
3. **确定交互**：标注每个页面的核心交互行为。
4. **构建原型**：按 Token 系统编写 HTML，实现所有交互。
5. **编写文档**：输出配套说明文档，覆盖交互、规则、边界。
6. **交付**：HTML 文件 + Markdown 文档一起交付。

## 质量检查

输出前检查：
- [ ] 所有按钮、链接是否可点击且有反馈？
- [ ] 页面间跳转关系是否正确？
- [ ] 表单是否可填写、提交是否有反馈？
- [ ] 弹窗/抽屉是否能打开和关闭？
- [ ] 列表数据是否足够真实（≥3 条）？
- [ ] 空状态是否设计？
- [ ] 灰度色阶是否统一（无意外彩色）？
- [ ] 配套文档是否覆盖所有交互说明？
- [ ] 文件是否可双击直接在浏览器打开？
