# 产品解决方案设计

## 方案设计结构

产品方案应回答：为什么做、给谁做、做什么、不做什么、怎么验证。

## 标准内容

1. **背景与问题定义**
   - 当前现象是什么？
   - 根因可能是什么？
   - 目标用户是谁？

2. **目标与指标**
   - 产品目标
   - 业务目标
   - 用户体验目标
   - 成功指标与护栏指标

3. **方案选择**
   - 低成本方案
   - 标准功能方案
   - 长期能力方案
   - 推荐主方案与取舍理由

4. **功能范围**
   - 本期包含
   - 本期不包含
   - 后续版本

5. **用户流程**
   - 入口
   - 关键步骤
   - 成功路径
   - 失败/异常路径

6. **业务规则**
   - 权限
   - 状态
   - 触发条件
   - 限制条件
   - 通知/提醒

7. **风险与依赖**
   - 技术依赖
   - 数据依赖
   - 跨团队依赖
   - 合规/隐私风险
   - 回滚方案

## MVP 拆分

MVP 应只验证最核心假设。写清楚：
- 核心假设
- 最小功能范围
- 人工/运营兜底方式
- 暂不做内容
- 验证指标
- 成功后扩展方向

## 原型输出

当方案涉及页面交互时，可输出 HTML 原型作为方案的可视化补充：

- **何时输出原型**：方案包含新页面、新交互流程、或需要向干系人直观展示方案时。
- **原型定位**：黑白灰线框风格，聚焦信息架构和交互逻辑，不涉及视觉设计。
- **交付物**：可交互的 HTML 文件 + 配套说明文档（交互说明、业务规则、边界情况）。
- **平台匹配**：根据需求确定目标平台（移动端/桌面端/平板），不默认。

原型是方案的一部分，不是独立产出。原型应能回答：
- 用户从哪个入口进入？
- 页面包含哪些信息模块？
- 核心操作路径是什么？
- 异常/空状态如何处理？
