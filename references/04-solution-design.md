# 产品解决方案设计

## 方案设计结构

产品方案应回答：为什么做、给谁做、做什么、不做什么、怎么验证。

## 标准内容

1. **背景与问题定义**
   - 当前现象是什么？
   - 根因可能是什么？
   - 目标用户是谁？

2. **目标与指标**
   - 产品目标
   - 业务目标
   - 用户体验目标
   - 成功指标与护栏指标

3. **方案选择**
   - 低成本方案
   - 标准功能方案
   - 长期能力方案
   - 推荐主方案与取舍理由

4. **功能范围**
   - 本期包含
   - 本期不包含
   - 后续版本

5. **用户流程**
   - 入口
   - 关键步骤
   - 成功路径
   - 失败/异常路径

6. **业务规则**
   - 权限
   - 状态
   - 触发条件
   - 限制条件
   - 通知/提醒

7. **风险与依赖**
   - 技术依赖
   - 数据依赖
   - 跨团队依赖
   - 合规/隐私风险
   - 回滚方案

## MVP 拆分

MVP 应只验证最核心假设。写清楚：
- 核心假设
- 最小功能范围
- 人工/运营兜底方式
- 暂不做内容
- 验证指标
- 成功后扩展方向
