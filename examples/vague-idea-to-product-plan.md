# 示例：模糊想法到产品方案

## 用户输入
我想做一个会员积分系统，帮我看看怎么做。

## 推荐处理

### 1. 先不要直接写功能
将“会员积分系统”还原成问题：
- 是为了提升复购？
- 提升会员活跃？
- 增加客单价？
- 降低流失？
- 还是增强用户粘性？

### 2. 关键澄清问题
1. 当前会员体系是否已有等级、权益或储值？
2. 希望积分影响哪个指标：复购、留存、客单价、裂变还是活跃？
3. 积分获取和消耗是否已有业务场景？
4. 是否有风控、防刷、财务成本约束？

### 3. 初步方案
- MVP：积分获取 + 积分余额 + 兑换基础权益。
- 暂不做：复杂等级、积分交易、积分商城、多任务体系。
- 核心指标：积分使用率、复购率、兑换转化率、权益成本率。

### 4. 输出形态
如果用户要讨论方向，输出产品解决方案；如果用户要交付研发，输出 PRD。
