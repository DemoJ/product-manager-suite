# 示例：PRD 审查

## 用户输入
帮我 review 这个 PRD：用户可以预约会议室，支持查看空闲状态、预约、取消和扫码签到。

## 推荐处理

### 1. 需求概述
这是一个会议室预约与签到功能，目标可能是提升会议室使用效率、减少冲突、沉淀使用数据。

### 2. 重点审查项
- 目标用户：员工、行政、会议组织者是否都考虑？
- 场景：预约、修改、取消、冲突、迟到、未签到、临时占用。
- 权限：谁能预约？是否有部门/时长/时间段限制？
- 状态：空闲、已预约、使用中、已取消、未签到释放。
- 异常：扫码失败、设备离线、重复预约、超时未签到。
- 指标：会议室使用率、预约冲突率、爽约率、扫码签到率。

### 3. 常见补充建议
- 增加预约冲突和释放规则。
- 明确取消和超时未签到机制。
- 增加管理员后台和数据统计。
- 定义通知规则：预约成功、会议前提醒、取消通知。

### 4. 输出形态
使用 `templates/requirement-review-report.md` 输出审查报告。
